# Onboarding

New to this repo? Start here. Three documents cover three angles on
"how do I work on this thing?"

## What to read first, based on what you're doing

**You want to set up the repo, the lab VM, and the stack locally** →
[`GETTING-STARTED.md`](GETTING-STARTED.md). Tools, versions, the
provisioning scripts, first-time gotchas.

**You want to understand why decisions look the way they do, or avoid
mistakes already made** → [`LESSONS-LEARNED.md`](LESSONS-LEARNED.md).
Empirical guidance on Podman, Jekyll, KVM, Aya, and working with AI
assistants on a long technical build.

**You want to continue the human-AI collaboration that produced this
tutorial** → [`STARTING-WITH-CLAUDE.md`](STARTING-WITH-CLAUDE.md). How
the back-and-forth is structured, and the resume prompt for a fresh
session.
